# Git ハンズオン Day 0：Gitをインストールする

## 今日のゴール

ハンズオン研修で使う **Git for Windows** をインストールして、正しく動くことを確認します。
所要時間は約20分です。研修当日までに完了させてください。

---

## 1. Git for Windows とは

Git for Windows は、Windows上でGitを使うためのソフトウェアです。
以下のツールがまとめてインストールされます。

| ツール名         | 説明                                                       |
| ---------------- | ---------------------------------------------------------- |
| **Git Bash**     | Gitコマンドを実行するためのターミナル（研修で主に使います） |
| **Git GUI**      | GUIでGitを操作するツール（今回は使いません）               |
| **Git CMD**      | WindowsのコマンドプロンプトからGitを使うためのツール       |

> **ポイント：** 研修では **Git Bash** を使います。
> Git Bash はLinuxと同じコマンドが使えるターミナルで、業界標準の操作方法を学べます。

---

## 2. ダウンロード（5分）

### 2-1. 公式サイトにアクセスする

```
1. ブラウザで https://gitforwindows.org/ にアクセスします
2. 画面中央の大きな「Download」ボタンをクリックします
```

<img src="https://gitforwindows.org/img/gwindows_logo.png" alt="Git for Windows 公式サイト" style="max-width: 25%; height: auto;" />

> **注意：** 必ず公式サイト（gitforwindows.org）からダウンロードしてください。
> 検索で出てくる非公式サイトからダウンロードしないでください。

### 2-2. インストーラーを保存する

```
ダウンロードが始まります。
「Git-2.xx.x-64-bit.exe」というファイルがダウンロードフォルダに保存されます。
```

> **ファイル名の数字（バージョン番号）はダウンロード時期によって異なります。**
> この資料は **2026年4月** に作成し、**Git for Windows 2.53.0.2** で動作を検証しています。
> それより新しいバージョンがダウンロードされた場合も、そのままインストールして問題ありません。

> **注意：** バージョンアップにより、次の「3. インストール」の画面構成や
> 選択肢の内容が変わる可能性があります。
> 画面が資料と異なる場合は、講師に確認してください。

## 3. インストール（10分）

### 3-1. インストーラーを起動する

```
1. ダウンロードした「Git-2.xx.x-64-bit.exe」をダブルクリックします
2. 「このアプリがデバイスに変更を加えることを許可しますか？」と聞かれたら「はい」を押します
```

### 3-2. ライセンスの確認

```
GNU General Public License の内容が表示されます。
「Next」をクリックします。
```

### 3-3. インストール先の選択

```
インストール先フォルダが表示されます。
変更せずそのまま「Next」をクリックします。
```

> **デフォルトのまま** で問題ありません。特に理由がなければ変更しないでください。

### 3-4. コンポーネントの選択

```
インストールする機能を選ぶ画面が表示されます。
デフォルトのチェックのまま「Next」をクリックします。
```

### 3-5. スタートメニューの設定

```
スタートメニューのフォルダ名が表示されます。
そのまま「Next」をクリックします。
```

### 3-6. デフォルトエディターの選択

```
Gitが使うテキストエディターを選ぶ画面が表示されます。
「Use Vim (the ubiquitous text editor)」が選ばれています。
そのまま「Next」をクリックします。
```

> **Vimって何？** Vimはターミナル上で動くテキストエディターです。
> 操作方法が独特ですが、研修で直接使うことはほとんどありません。
> もし間違ってVimが開いてしまった場合は、`:q` と入力してEnterキーを押すと閉じられます。

### 3-7. 新しいリポジトリのブランチ名

```
「Override the default branch name for new repositories」を選択し、
「main」と入力されていることを確認して「Next」をクリックします。
```

> **なぜ main にするのか？**
> GitHubではデフォルトのブランチ名が `main` です。
> ここを合わせておくと、後の操作でトラブルが減ります。

### 3-8. 環境変数（PATH）の設定

```
「Git from the command line and also from 3rd-party software」
（推奨オプション）が選ばれていることを確認して「Next」をクリックします。
```

### 3-9. SSHの設定

```
「Use bundled OpenSSH」が選ばれていることを確認して「Next」をクリックします。
```

### 3-10. HTTPSの設定

```
「Use the OpenSSL library」が選ばれていることを確認して「Next」をクリックします。
```

### 3-11. 改行コードの設定

```
「Checkout Windows-style, commit Unix-style line endings」
が選ばれていることを確認して「Next」をクリックします。
```

> **改行コードとは：**
> WindowsとLinux/macOSでは、テキストファイルの「改行」の表現方法が異なります。
> この設定にしておくと、チームメンバーがどのOSを使っていても問題が起きにくくなります。

### 3-12. ターミナルの設定

```
「Use MinTTY (the default terminal of MSYS2)」が選ばれていることを確認して
「Next」をクリックします。
```

### 3-13. git pull の動作設定

```
「Default (fast-forward or merge)」が選ばれていることを確認して
「Next」をクリックします。
```

### 3-14. 資格情報マネージャーの設定

```
「Git Credential Manager」が選ばれていることを確認して
「Next」をクリックします。
```

> **Git Credential Manager とは：**
> GitHubにpushするとき、毎回パスワードを入力しなくて済むようにしてくれる仕組みです。
> 初回だけブラウザで認証すれば、2回目からは自動でログインされます。

### 3-15. その他のオプション

```
「Enable file system caching」にチェックが入っていることを確認して
「Next」をクリックします。
```

### 3-16. 実験的オプション

```
何もチェックを入れずに「Install」をクリックします。
```

> **「Experimental」と書かれた機能** は、まだ開発中の機能です。
> 安定して動く保証がないので、チェックしないでください。

### 3-17. インストール完了

```
インストールが完了すると「Completing the Git Setup Wizard」と表示されます。
「Launch Git Bash」にチェックを入れて「Finish」をクリックします。
```

## 4. 動作確認（5分）

### 4-1. Git Bash が開くことを確認する

```
黒いターミナル画面（Git Bash）が開いていればOKです。
閉じてしまった場合は、スタートメニューから「Git Bash」を検索して開いてください。
```

### 4-2. バージョンを確認する

Git Bash に以下のコマンドを入力して、Enterキーを押します。

```
git --version
```

以下のようにバージョン番号が表示されればインストール成功です。

```
git version 2.xx.x.windows.x
```

> **数字はインストールした時期によって異なります。** 表示されれば問題ありません。

### 4-3. 基本コマンドを試す

以下のコマンドも試してみましょう。

```
pwd
```

> 今いるフォルダのパスが表示されます（例：`/c/Users/あなたの名前`）

```
ls
```

> フォルダの中身が一覧で表示されます

```
cd Desktop
```

> デスクトップに移動します

```
pwd
```

> `/c/Users/あなたの名前/Desktop` と表示されればOKです

## 5. トラブルシューティング

### 「git」と入力しても反応がない

Windowsのコマンドプロンプト（黒い画面）ではなく、**Git Bash** を使っているか確認してください。
スタートメニューで「Git Bash」と検索して起動してください。

### インストーラーが起動しない

「このアプリがデバイスに変更を加えることを許可しますか？」で「いいえ」を押していないか確認してください。
管理者権限が必要な場合は、講師に相談してください。

### ダウンロードがブロックされる

会社のセキュリティソフトがダウンロードをブロックする場合があります。
講師に相談してください。
